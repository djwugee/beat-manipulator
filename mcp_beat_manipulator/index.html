Research Findings

Model Context Protocol (MCP) Overview

MCP is an open protocol that standardizes how AI applications communicate with external tools and data sources. It provides:

Standardized Interface

JSON-RPC 2.0 based communication with standardized tool discovery and execution mechanisms.

Security Boundaries

Clear separation between AI applications and external tools with proper capability negotiation.

Tool Composition

Multiple tools can be combined seamlessly to create complex workflows.

Beat Manipulator Repository Analysis

The beat-manipulator repository provides advanced AI-powered beat manipulation capabilities:

Multi-song Mixing Mix multiple songs using patterns mix_multiple_songs

class BeatManipulatorMCPServer:
    def __init__(self):
        self.server = Server("beat-manipulator-mcp")
        self.setup_tools()
        
    def setup_tools(self):
        @self.server.list_tools()
        async def handle_list_tools() -> List[Tool]:
            return [
                Tool(
                    name="beatswap",
                    description="Swap beats in an audio file using a pattern",
                    inputSchema={...}
                ),
                # ... other tools
            ]
        
        @self.server.call_tool()
        async def handle_call_tool(name: str, arguments: dict):
            if name == "beatswap":
                return await self.handle_beatswap(arguments)
            # ... other handlers

Tool Implementation Example

async def handle_beatswap(self, arguments: dict) -> List[TextContent]:
    audio = arguments.get("audio")
    pattern = arguments.get("pattern")
    scale = arguments.get("scale", 1.0)
    shift = arguments.get("shift", 0.0)
    
    try:
        # Load audio and generate beatmap
        song = bm.song(audio=audio)
        song.beatmap_generate()
        
        # Apply transformations
        song.beatmap_shift(shift)
        song.beatmap_scale(scale)
        
        # Perform beat swapping
        result = song.beatswap(pattern=pattern)
        
        # Save output
        song.write(output=output)
        
        return [TextContent(
            type="text", 
            text=f"Successfully beatswapped audio using pattern '{pattern}'"
        )]
    except Exception as e:
        return [TextContent(type="text", text=f"Error: {str(e)}")]

MCP Tools Specification

The MCP server exposes 5 core tools that map to the beat-manipulator's functionality:

beatswap

Rearrange beats in audio using pattern syntax. Supports scale, shift, and custom output paths.

{
  "audio": "path/to/song.mp3",
  "pattern": "1,3,2,4",
  "scale": 1.0,
  "shift": 0.0,
  "output": "output.wav"
}

generate_beatmap

Generate beatmap using AI-powered beat detection with caching and variable BPM support.

{
  "audio": "path/to/song.mp3",
  "caching": true,
  "variable_bpm": false
}

slice_audio_by_beats

Extract specific beats or beat ranges from audio files with precision timing.

{
  "audio": "path/to/song.mp3",
  "beat_start": 8,
  "beat_end": 16
}

apply_audio_effects

Apply audio effects using pattern syntax: speed, reverse, volume, bitcrush, downsample.

{
  "audio": "path/to/song.mp3",
  "pattern": "1s2r,2d8,3v0.5"
}

mix_multiple_songs

Mix multiple songs using beat manipulation patterns with [song2] syntax.

{
  "primary_audio": "song1.mp3",
  "secondary_audio": "song2.mp3",
  "pattern": "1,[song2]2,3,[song2]4"
}

Pattern Syntax Reference

Syntax Description Example
, Sequential playback "1,2,3,4"
; Simultaneous mixing "1;2"
s Speed effect "1s2" (2x speed)
r Reverse effect "1r"
v Volume effect "1v0.5" (50% volume)
d Downsample effect "1d8" (8-bit)
b Bitcrush effect "1b4"
! Skip beat "1,2,3,4!"
@ Random selection "@1_4_0.5"

Deployment Options

1. Local Installation

# Install dependencies
pip install -r requirements.txt
pip install git+https://github.com/CPJKU/madmom

# Install MCP server
pip install -e .

# Run the server
mcp-beat-manipulator

2. Claude Desktop Integration

Add to your Claude Desktop configuration:

{
  "mcpServers": {
    "beat-manipulator": {
      "command": "mcp-beat-manipulator",
      "env": {
        "PYTHONPATH": "/path/to/beat-manipulator"
      }
    }
  }
}

3. Docker Deployment

# Build and run with Docker
docker-compose up -d

# Or manually
docker build -t mcp-beat-manipulator .
docker run -v ./audio:/app/audio mcp-beat-manipulator

4. Development Setup

# Clone and setup
git clone 
cd mcp-beat-manipulator

# Create virtual environment
python -m venv venv
source venv/bin/activate  # or venv\Scripts\activate on Windows

# Install in development mode
pip install -e .

Usage Examples

Example 1: Basic Beat Swapping

# Swap 2nd and 3rd beats every 4 beats
mcp-client call-tool beat-manipulator beatswap --arguments '{
  "audio": "/path/to/song.mp3",
  "pattern": "1,3,2,4"
}'

Example 2: Advanced Effects

# Apply multiple effects to different beats
mcp-client call-tool beat-manipulator apply_audio_effects --arguments '{
  "audio": "/path/to/song.mp3",
  "pattern": "1s2r,2d8,3v0.5,4b4"
}'

Example 3: Multi-song Mixing

# Mix two songs alternating beats
mcp-client call-tool beat-manipulator mix_multiple_songs --arguments '{
  "primary_audio": "song1.mp3",
  "secondary_audio": "song2.mp3",
  "pattern": "1,[song2]2,3,[song2]4"
}'

Example 4: Audio Slicing

# Extract beats 8-16 from a song
mcp-client call-tool beat-manipulator slice_audio_by_beats --arguments '{
  "audio": "/path/to/song.mp3",
  "beat_start": 8,
  "beat_end": 16
}'

Example 5: Generate Beatmap

# Generate beatmap with caching enabled
mcp-client call-tool beat-manipulator generate_beatmap --arguments '{
  "audio": "/path/to/song.mp3",
  "caching": true,
  "variable_bpm": false
}'
